## 🎨 Component Architecture (OOP Principles)

### 1. Single Responsibility Principle (SRP)
- Each component has ONE job
- PokemonCard: Display Pokemon
- PokemonFilters: Handle filtering
- Pagination: Handle page navigation

### 2. Open/Closed Principle (OCP)
- Components open for extension, closed for modification
- Use composition over inheritance
- Props for customization

### 3. Liskov Substitution Principle (LSP)
- Base components can be replaced by variants
- Button → PrimaryButton, SecondaryButton
- Card → PokemonCard, MetricsCard

### 4. Interface Segregation Principle (ISP)
- Small, focused interfaces (TypeScript types)
- No "god" interfaces
- Specific props for each component

### 5. Dependency Inversion Principle (DIP)
- Depend on abstractions (hooks, services)
- Not on concrete implementations
- Inject dependencies via props/context

## 📦 Reusable Components Strategy

### Atomic Design Pattern

**Atoms** (ui/)
- button, input, label, badge
- Smallest building blocks
- No business logic

**Molecules** (common/)
- SearchBar (input + button)
- ErrorMessage (icon + text)
- LoadingSpinner (animation)

**Organisms** (pokemon/, health/)
- PokemonCard (image + stats + badges)
- PokemonTable (table + pagination + filters)
- HealthDashboard (multiple cards + charts)

**Templates** (layout/)
- DashboardLayout (nav + sidebar + content)
- AuthLayout (centered form)

**Pages** (app/)
- Compose templates + organisms
- Handle routing and data fetching

## 🔄 State Management Strategy

### Zustand Slices (Modular)

**authSlice**
- user, token, isAuthenticated
- login(), logout(), checkAuth()

**pokemonSlice**
- pokemon[], filters, pagination
- fetchPokemon(), setFilters()

**healthSlice**
- systemHealth, jobs, metrics
- fetchHealth(), subscribeToJobs()

**uiSlice**
- theme, sidebarOpen, notifications
- toggleTheme(), showNotification()

**wsSlice**
- connected, reconnecting, error
- connect(), disconnect(), send()

## 🚀 Key Features

### 1. Authentication Flow
- Login → JWT token → Store in httpOnly cookie
- Middleware checks auth on protected routes
- Auto-refresh token before expiry
- Secure logout (clear token + redirect)

### 2. Pokemon List
- Server-side pagination (page, limit)
- Client-side filtering (type, name)
- Sorting (power_score, name)
- Loading skeletons
- Error handling with retry

### 3. System Health Dashboard
- Real-time job status (WebSocket)
- Metrics cards (total Pokemon, active jobs)
- Progress indicators
- Job history table
- Auto-refresh every 30s (fallback)

### 4. Security Features
- CSRF protection on forms
- XSS prevention (sanitize inputs)
- Rate limiting (max 100 req/min)
- Content Security Policy
- Secure headers

### 5. Performance Optimization
- Code splitting (dynamic imports)
- Image optimization (Next.js Image)
- Lazy loading components
- Debounced search
- Memoized expensive calculations
